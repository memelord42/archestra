‚Źļ Looking at the spike, "prompt-on-install" fields today are FieldDef with kind:
   "user" + source: "prompt-install" (Jira username+api_token, Vertica
  username+password, Context7 context7_api_key). The install dialog renders one
  input per field.

  A clean way to add presets is to mirror what Environment already does for env
  fields ‚ÄĒ make a Preset record that stores values for some/all of a catalog's
  prompt-install fields, scoped (personal/team/org) and shown in the Install
  dialog as a dropdown that pre-fills the inputs (still editable). Tradeoff:
  presets containing secrets need the same secret-storage path as credentials,
  and you double the "what do I pick?" UX in the dialog (Environment + Preset
  selectors side by side).

  Two design questions before I touch code:
  1. Scope ‚ÄĒ personal-only (shortcut for me), team/org-shared (shared service
  accounts), or both?
  2. All vs subset ‚ÄĒ does a preset have to fill every prompt-install field, or
  can it cover a subset and leave the rest to be typed (e.g. preset fills
  api_token, user types own username)?
